# Architecture Patterns

**Domain:** Undervisningsmoment - Kallkritik, konspirationsteorier och AI (gymnasiet)
**Researched:** 2026-03-10

## Recommended Architecture

The module follows a three-phase arc: **Foundation -> Application -> Synthesis**. This maps to Rosenshine's scaffolding principle: provide heavy support early, remove it systematically, and require independent performance before assessment.

```
Phase 1: FOUNDATION (Lektion 1-3)        Phase 2: APPLICATION (Lektion 4-5)       Phase 3: SYNTHESIS (Lektion 6-7+exam)
Build the toolkit                         Use tools with decreasing support         Demonstrate mastery
High scaffolding                          Scaffolding removal explicit              Independent performance
Single-concept focus per lesson           Multi-concept integration                 Full integration + reflection

L1: Source criticism      L2: Conspiracy     L3: AI+Conspiracy     L4: Seminar      L5: Written        L6: Summative      L7: Reflection
    + AI basics               + Harvard          combined              (oral)           analysis           assessment         + self-eval
    intro                     intro                                                    (written)          (written)
```

### Lesson Progression Logic

The progression has three critical transitions that determine module success:

**Transition 1: L1->L2 (AI focus -> Conspiracy focus)**
Smooth. Both use the same four source criticism questions as analytical framework. Students apply familiar tools to new content. Harvard referencing introduced here (per change plan) creates a running thread.

**Transition 2: L3->L4 (Toolkit -> Application) - THE CRITICAL JUNCTION**
This is where scaffolding drops. The devil's advocate review correctly identified this as under-prepared. The architecture must include an explicit "gateway check" - a formative moment at the end of L3 or start of L4 that signals to both teacher and students: "You now have the tools. From here, you use them independently."

Concrete mechanism: L3 exit ticket serves as gateway. If students cannot combine AI + conspiracy theory analysis without the matrix scaffold, L4 needs adjusted scaffolding levels.

**Transition 3: L5->L6 (Practice -> Assessment)**
The written perspective analysis in L5 (per change plan: replacing the debate) serves as a dress rehearsal for L6's summative assessment. Same skill (written analysis with sources), lower stakes. This is architecturally sound - it follows the "I do, We do, You do" principle where L5 = "We do" (with peer support, role cards available) and L6 = "You do" (summative, independent).

### Component Boundaries

| Component | Responsibility | Feeds Into |
|-----------|---------------|------------|
| **L1: Kan du lita pa det du ser?** | Introduce 4 source criticism questions + AI-generated content recognition | L2 (retrieval of questions), L3 (AI analysis skills) |
| **L2: Vem tjanar pa att du tror det har?** | Conspiracy theory mechanics + Harvard referencing intro | L3 (conspiracy analysis), L4-6 (Harvard skill) |
| **L3: Verktygsladan i praktiken** | Combine AI + conspiracy, apply full toolkit | L4 (readiness for independent application) |
| **L4: Informationskriget** | Oral application - seminar with positions | L5 (arguments to develop in writing) |
| **L5: Perspektiven krockar** | Written perspective analysis with Harvard refs | L6 (writing skill rehearsal) |
| **L6: Fran analys till text** | Summative written assessment with Harvard refs | L7 (reflection on learning) |
| **L7: Vad har vi lart oss?** | Synthesis, reflection, self-assessment | N/A (module end) |
| **L8: Seminarium (exam)** | Summative oral examination | N/A |

### Data Flow: Formative -> Summative Pipeline

```
L1 exit ticket -----> L2 retrieval practice (adjusts based on gaps)
L2 exit ticket -----> L3 retrieval practice
L3 exit ticket -----> L4 retrieval practice + GATEWAY CHECK
L4 exit ticket -----> L5 retrieval practice (strongest argument from seminar -> written)
L5 exit ticket -----> L6 retrieval practice
L6 summative -------> L7 reflection (students see their own growth)
L6 summative -------> L8 seminar exam (teacher knows student levels)
L7 self-assessment -> L8 seminar (students know their strengths/gaps)
```

This pipeline is the module's backbone. Every exit ticket serves dual purpose: (1) formative feedback for teacher to adjust next lesson's retrieval practice, and (2) spaced repetition for students.

## Patterns to Follow

### Pattern 1: Spiral Harvard Referencing
**What:** Introduce Harvard in L2 (basic - "write one reference"), practice in L3 exit ticket, require in L5 written analysis, assess in L6 summative.
**When:** Any skill that needs distributed practice across a module.
**Why:** Skills taught once and tested three lessons later decay. The spiral ensures 4 touchpoints before summative assessment.

```
L2: See it (intro, 10 min)
L3: Try it (exit ticket requires one reference)
L5: Practice it (written analysis, 2+ sources required)
L6: Prove it (summative, correct referencing assessed)
```

### Pattern 2: Mode Alternation (Oral -> Written -> Oral)
**What:** Alternate between oral and written expression across lessons to prevent fatigue and reach different learners.
**When:** Multi-lesson modules with both discussion and writing goals.

```
L1: Hands-on + discussion (mixed)
L2: Group analysis + presentation (oral-heavy)
L3: Combined analysis (mixed)
L4: Seminar (oral)
L5: Written perspective analysis (written)
L6: Written summative (written)
L7: Reflection + discussion (oral/mixed)
L8: Seminar exam (oral)
```

The change plan's decision to replace L5 debate with written analysis is architecturally correct - it breaks the oral-oral sequence (L4-L5) and creates a written-written preparation sequence (L5-L6) that serves assessment readiness.

### Pattern 3: Scaffolding Fade with Explicit Triggers
**What:** Define specific criteria for when scaffolding is removed, not just "gradually from L4."
**When:** Any progression from supported to independent work.

```
L1-2: Full scaffolding (matrices, sentence starters, model answers)
L3:   Partial scaffolding (matrix available but not pre-filled, no sentence starters)
L4:   Scaffolding on request (materials available but not distributed by default)
L5:   Optional scaffolding (role cards available for E-level, not for A-level)
L6:   Minimal scaffolding (Harvard cheat-sheet only)
L7:   No scaffolding (free reflection)
```

**Trigger for removal:** If a student's exit ticket demonstrates the skill without using the scaffold, remove it next lesson. If not, keep it one more lesson.

### Pattern 4: Conspiracy Theories with Intellectual Respect
**What:** Frame conspiracy theories as understandable human responses to uncertainty, not as stupidity. Analyze mechanisms (pattern-seeking, distrust of institutions, need for agency) before evaluating truth claims.
**When:** Any lesson touching conspiracy content.
**Why:** The UNESCO guidelines and media literacy research consistently show that dismissive approaches backfire - students who feel judged for their beliefs (or their family's beliefs) disengage. Understanding WHY people believe is the source-critical skill; labeling beliefs as "wrong" is not.

```
Sequence:
1. "What makes this convincing?" (understand appeal)
2. "What mechanisms are at work?" (analyze spread)
3. "How would we verify this?" (apply source criticism)
4. "What are the consequences?" (evaluate impact)

NOT: "This is wrong because..."
```

## Anti-Patterns to Avoid

### Anti-Pattern 1: Overloaded Pivot Lessons
**What:** Packing too many learning goals into the lesson that bridges phases.
**Why bad:** L3 in the original plan had 4 learning goals in 75 minutes - the shortest lesson in the foundation phase. The bridge lesson needs MORE time, not less, because it integrates previously separate concepts.
**Instead:** Keep bridge lessons (L3) focused on integration of existing skills. Move new skills (Harvard) to earlier lessons. Maximum 2-3 learning goals per lesson.

### Anti-Pattern 2: Invisible Scaffolding Removal
**What:** Telling the teacher "scaffolding decreases from L4" without specifying what, when, and for whom.
**Why bad:** Teachers either remove everything at once (students flounder) or keep everything (students never become independent). The devil's advocate review flagged this correctly.
**Instead:** Specify per-lesson what scaffolding is available, what is removed, and what the trigger criteria are.

### Anti-Pattern 3: Formative-Summative Gap
**What:** Building a strong formative assessment system (exit tickets every lesson) but disconnecting it from the summative assessment.
**Why bad:** Exit tickets measure source criticism understanding; the summative assesses written analysis with Harvard referencing. If formative data never covers the summative skills, the teacher has no early warning.
**Instead:** Ensure at least 2 exit tickets (L3, L5) include Harvard referencing practice so the teacher can identify struggling students before L6.

### Anti-Pattern 4: Debate Fatigue
**What:** Scheduling two oral argumentation activities (seminar + debate) on consecutive days.
**Why bad:** Introverted students suffer twice. Dominant students dominate twice. The pedagogical value of the second session is marginal. The change plan correctly identified this.
**Instead:** Alternate modes. Use oral in L4, written in L5. The written format also gives quieter students a more equal platform and creates a document trail the teacher can assess.

## Architecture Implications for Build Order

### Recommended Material Creation Sequence

The build order should follow dependency logic, not lesson number:

**Build Phase 1: Structural Foundation**
1. **Momentplan (updated)** - This is the master document. All lesson plans depend on it. Must reflect the three changes from the devil's advocate review.
2. **Assessment criteria (E/C/A)** - Must be defined before lesson plans because they determine what each lesson aims toward.

**Build Phase 2: Bookends First**
3. **L1 (opening)** - Sets the tone, introduces core framework. Already largely done but needs updates for current examples.
4. **L6 (summative assessment)** - Define the destination before building the journey. What exactly must students demonstrate? This determines what L2-L5 must teach.
5. **L8 (seminar exam)** - Define oral examination criteria. These inform L4's seminar structure.

**Build Phase 3: Foundation Lessons**
6. **L2** - Conspiracy theory mechanics + Harvard intro (depends on knowing what Harvard skills L6 requires)
7. **L3** - Integration lesson (depends on L1 and L2 content being finalized)

**Build Phase 4: Application Lessons**
8. **L4** - Seminar (depends on L3 content and L8 exam format for alignment)
9. **L5** - Written perspective analysis (depends on L4's seminar questions for continuity)

**Build Phase 5: Closing**
10. **L7** - Reflection (depends on all other lessons being finalized - it references the whole journey)
11. **Momentoversikt (HTML)** - Student-facing overview (depends on all lessons being finalized)

**Rationale for this order:**
- Momentplan first because everything depends on it
- Assessment (L6, L8) before instruction (L2-L5) because backward design produces better alignment
- L1 early because it is already partially built and sets the framework
- L7 and HTML last because they summarize/reference everything else

## Theory-Practice Balance

| Lesson | Theory Load | Practice Load | Balance |
|--------|-------------|---------------|---------|
| L1 | 4 source criticism questions (theory) | Sorting exercise + AI lab (practice) | 40/60 |
| L2 | Conspiracy mechanisms, spread patterns (theory) | Case analysis, spread mapping (practice) | 35/65 |
| L3 | AI + conspiracy intersection (theory) | Group analysis combining both (practice) | 30/70 |
| L4 | Democracy + disinformation (theory) | Seminar discussion (practice) | 20/80 |
| L5 | Perspective framing (minimal theory) | Written analysis (practice) | 10/90 |
| L6 | Harvard recap (minimal) | Summative writing (practice) | 5/95 |
| L7 | None | Reflection + self-assessment | 0/100 |

**Pattern:** Theory front-loaded in L1-L3, practice dominates L4-L7. This is correct - you cannot apply what you have not learned, and you cannot demonstrate mastery through theory alone. The transition from 40/60 to 5/95 should be smooth, not abrupt.

## Formative-Summative Connection Architecture

The module uses a **continuous formative pipeline feeding two summative endpoints:**

```
FORMATIVE STREAM                          SUMMATIVE ENDPOINTS
------------------                        -------------------
L1 exit ticket (source criticism Qs)
L2 exit ticket (conspiracy analysis)  -->  Reference skill tracking
L3 exit ticket (combined + Harvard)   -->  Gateway check for Phase 2
L4 exit ticket (argument quality)     -->  Seminar skills preview
L5 exit ticket (perspective shift)    -->  Writing readiness check
                                           |
                                           v
                                      L6: Written assessment (Summative 1)
                                           |
                                           v
                                      L7: Self-assessment + reflection
                                           |
                                           v
                                      L8: Seminar exam (Summative 2)
```

**Key design principle:** The teacher should have NO surprises at L6 or L8. By the time students reach summative assessment, the formative stream has already revealed their level. If exit tickets from L3 and L5 show a student cannot reference sources correctly, the teacher intervenes before L6 - not after.

## Scalability Considerations

| Concern | Small group (15) | Standard class (25) | Large class (30+) |
|---------|-------------------|----------------------|--------------------|
| Seminar L4 | One group, rich discussion | Split into 2 seminars, teacher rotates | 3+ groups needed, requires structured protocol |
| Presentations L2 | All groups present | 2-3 present, others give written feedback | Written feedback only, select 2 for oral |
| Exit ticket analysis | Read all in 10 min | Sort into 3 piles (15 min) | Digital exit tickets (Google Forms) essential |
| Written assessment L6 | Full formative feedback | Focused feedback on 2 criteria | Peer feedback first, teacher confirms grades |
